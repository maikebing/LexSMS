# 诊断工具使用说明

## 概述
`DiagnosticExample` 是一个用于诊断 LexSMS 模块初始化问题的工具。它会显示详细的初始化日志和错误信息，帮助您快速定位问题。

## 使用方法

1. 修改 `Program.cs` 中的串口名称：
```csharp
string portName = "COM3"; // Windows: COM3, Linux: /dev/ttyUSB0
```

2. 运行程序：
```bash
cd samples/DiagnosticExample
dotnet run
```

## 常见问题

### 1. SIM 卡未插入 (`+CME ERROR: SIM not inserted`)
**原因：** SIM 卡未正确插入或未被模块识别

**解决方法：**
- 检查 SIM 卡是否正确插入卡槽
- 确保 SIM 卡金属触点清洁
- 尝试重新插拔 SIM 卡
- 检查 SIM 卡大小是否与卡槽匹配

### 2. 需要 PIN 码 (`+CME ERROR: SIM PIN required`)
**原因：** SIM 卡启用了 PIN 码保护

**解决方法：**
- 使用手机关闭 SIM 卡 PIN 码
- 或在代码中添加 PIN 码验证逻辑

### 3. 无法注册网络
**原因：** 模块无法连接到移动网络

**解决方法：**
- 检查天线是否正确连接
- 检查当前位置的信号强度（远离地下室、电梯等）
- 确认 SIM 卡已激活且未欠费
- 确认 SIM 卡运营商在当前地区有信号覆盖
- 尝试更换位置重试

### 4. 串口打开失败
**原因：** 无法打开指定的串口

**解决方法：**
- 检查串口名称是否正确（Windows: COM3, Linux: /dev/ttyUSB0）
- 确认串口未被其他程序占用
- 检查串口驱动是否正确安装
- 检查当前用户是否有串口访问权限（Linux 需要加入 dialout 组）
